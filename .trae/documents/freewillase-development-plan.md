## 1. 建设目标
本阶段开发目标是交付一个可用于内部试运行的 FreeWillase 平台，优先实现基于 NCBI 的酶数据库自动构建、文献自动 Match 入口、结构与文献公开元数据展示，形成科研团队可持续使用的最小可用平台。
- 第一优先级是把 accession 批量导入、NCBI 自动补全、酶条目建库打通，形成稳定的数据底座。
- 第二优先级是建设“自动 Match”按钮和文献关联结果页，让酶条目可以自动关联候选文献。
- 第三优先级是预留 `MiniFold` 预测接口与参数模型，为后续算法接入保留扩展位，但首期不实际开发。

## 2. 范围界定
### 2.1 本期范围
- 酶基础信息管理、序列版本管理、结构记录管理
- 基于 NCBI 的 accession 批量导入与自动建库
- 本地结构文件引用、外部结构链接、三维展示与下载
- 基础功能注释与人工维护
- 文献关联、结构域管理、功能位点管理、自动 Match 按钮
- 数据集导入与结果留痕
- 系统用户、角色、模型配置、日志审计

### 2.2 暂不纳入首期
- 复杂分子对接与反应路径模拟
- 大规模公共数据库全量镜像同步
- 自动化实验设备联动

## 3. 实施策略
平台采用“先业务闭环、后算法增强；先服务化封装、后高性能扩展”的策略推进。
- 业务闭环优先：先完成 accession 批量导入、酶条目建库、Match 按钮、结构与文献展示四个核心链路，再扩展外部接口与高级分析。
- 算法预留优先：先把 `MiniFold` 的任务模型、接口协议、参数模板预留好，再进入真正的算法服务化阶段。
- 标准化优先：序列、结构、任务、注释、日志全部采用统一数据模型和状态机管理。
- 可追溯优先：所有 accession 导入结果、NCBI 来源字段、文献证据、外部链接、接口配置和文件路径必须落库。

- 增加 Match 规则打分、强弱关联分层和关联依据展示。
- 加入基础统计报表、结构来源分布、文献关联数量、导入成功率和模块使用情况等运维视图。

### 阶段五：联调、验收与部署（第 14-16 周）

## 7. MiniFold 预留方案
### 7.1 首期处理原则
- 首期不开发 `MiniFold` 真正集成能力，不建设 Python 推理服务。
- 仅在数据库、配置页、接口协议和前端页面上预留算法接入位置。
- 先把结构结果查看页、参数模板和接口说明做出来，后续可平滑切换到真实执行。

### 7.2 推荐预留内容
1. 预留预测引擎注册表，支持 `MINIFOLD`、`ESM`、`CUSTOM` 等枚举。
2. 预留预测任务数据结构，但首期仅允许查看和配置，不允许实际提交执行。
3. 预留回调协议、结果文件协议、状态机定义和参数模板。
4. 在系统管理中配置接口地址、启用状态、认证方式、超时阈值等字段。

### 7.3 与现有 MiniFold 模块的映射建议
| 现有模块 | 现状 | 平台化改造建议 |
|----------|------|----------------|
| `pipeline.py` | 脚本式流程编排 | 后续接入时作为服务编排参考，首期仅保留说明 |
| `ark_module.py` | 直接读环境变量调用外部接口 | 后续抽象为配置化接口，首期不开发 |
| `backbone_predictor.py` | 计算逻辑集中 | 后续作为主链折叠核心，首期仅预留字段 |

## 8. Java 开发要点
- 所有后台作业采用明确状态机：`PENDING -> RUNNING -> SUCCESS/FAILED/CANCELED`。
- 所有外部接口预留协议要求幂等，避免后续接入时回调重复入库。
- 批量 accession 导入需要单独的任务主表与明细表，支持失败重试、重复跳过和断点恢复。
- NCBI 结果必须做标准化映射，以 `Protein accession.version + sequence hash + tax_id` 作为核心去重依据。
- `references` 字段只作为候选召回来源之一，最终文献展示必须回查 PMID/DOI 标准元数据。
- 文件下载统一经 Spring Boot 鉴权后转发，不直接暴露磁盘路径。
- 重要接口需要接口权限与操作日志，尤其是模型配置、数据导入、文献关联配置。
- 对长耗时结构解析或批量导入提供超时阈值、手动取消、失败重试与日志追踪。

## 9. 前端开发要点
- 采用统一布局框架与通用查询表格组件，保证列表型页面开发效率。
- accession 导入页需支持文本输入、Excel/CSV 上传、模板下载、任务进度和导入报告。
- 结构结果查看页使用“3D 视图 + 质量评分 + 结构域/位点 + 关联文献”四区联动布局。
- 文献模块需支持表格检索、详情抽屉、DOI 跳转和关系图式展示。
- 酶详情页需提供醒目的“自动 Match”按钮，并展示当前 Match 状态、结果数量和关联依据。
- 预测接口预留页采用禁用态或说明态交互，避免用户误以为首期已具备预测执行能力。
- 对结构和注释详情提供导出能力，支持 Markdown、JSON、PDB 资源下载。

## 10. Python 预留说明
- 首期不编写正式 Python 服务，也不部署独立 Python 运行时。
- 如果后续需要接入 `MiniFold`，Python 仅负责科学计算和模型推理，不负责用户、权限、条目 CRUD、文献关系管理等业务逻辑。
- 后续若扩展文献抽取算法，也建议把 Python 限制在“文本抽取 / 算法评分 / 结构预测”这类计算型能力。
- Java 始终保留主业务编排权，Python 只作为能力服务被调用。

## 11. 数据治理与标准
- 酶条目主键采用平台内部编码，不依赖外部数据库原始 ID。
- accession 作为建库主输入，优先要求 `protein accession.version`，缺少版本号时需做补全和风险提示。
- FASTA 序列采用版本化管理，并记录哈希值，防止重复或覆盖。
- 功能注释必须区分来源：人工、预测预留、导入，并保留证据等级。
- 文献关联必须能定位到酶、结构、结构域或功能位点中的至少一种对象。
- 结构结果必须记录来源方式：本地文件、外部链接、预测预留。
- 文献展示默认只保留公开元数据：标题、作者、期刊、年份、摘要、DOI、PMID、PMCID 和外链。
- 数据导入前先做模板校验、主键冲突校验、枚举值校验和长度校验。

## 13. 验收标准
- 用户可以完成 accession 批量导入建库、酶条目查看、结构记录维护、文献自动 Match、结果查看、注释维护的完整闭环。
- 结构结果全流程具备可见来源、文件留痕、结果可查、结果可下载能力。
- NCBI 导入任务具备成功数、失败数、重复数、跳过数和失败原因报告。
- 文献关联页面只展示公开元数据和外部链接，且能区分强关联、弱关联、候选关联。
- `MiniFold` 至少完成一套明确的预留接口定义、参数模板和前端配置页。
- 关键列表与详情页操作响应符合内部试运行要求，普通查询接口平均响应时间控制在 2 秒内。
- 管理员可以独立维护用户、角色、模型配置和导入模板。

## 14. 风险与应对
| 风险 | 说明 | 应对策略 |
|------|------|----------|
| 外部结构数据不一致 | 不同来源结构编号和字段差异大 | 统一结构来源字段和链接规范 |
| 文献元数据质量不稳定 | DOI、作者、摘要信息可能缺失 | 增加必填校验和人工补录机制 |
| 预留接口被误解为可用功能 | 用户可能认为预测已经完成 | 在页面和文档中明确“预留未启用”状态 |
| NCBI 返回字段不稳定 | 不同条目记录完整度不同 | 采用标准化映射层和缺省值填充策略 |
| accession 批量导入失败率高 | 输入错误或版本过期会造成批量失败 | 增加格式校验、任务报告和失败重试能力 |
| 数据标准不统一 | 不同来源酶数据字段差异大 | 先定义字段字典、模板规则和人工维护规范 |
| 结果解释性不足 | 预测分数与注释结果不易被科研人员信任 | 保留参数快照、评分明细、日志摘要和证据链展示 |

## 15. 部署建议
### 15.1 开发环境
- 前端、Spring Boot、本地 MySQL 分别运行，Spring Boot 直接调用 NCBI 接口，方便快速迭代。

### 15.2 测试环境
- 使用单独测试库和本地结构样例数据，验证 accession 自动建库、结构与文献链路。

### 15.3 生产环境
- `Nginx + Vue 静态资源 + Spring Boot 主服务 + MySQL`
- 文件存储可先使用本地共享目录，数据规模扩大后迁移到 MinIO。
- 未来接入 `MiniFold` 时，再单独增加 Python 推理节点。

## 16. NCBI 自动建库与 Match 路线
### 16.1 自动建库最小闭环
1. 用户批量提交 `protein accession.version`
2. 系统创建导入任务并校验格式
3. 调用 NCBI `Protein` 获取主记录
4. 通过 `ELink` 补齐 `Gene`、`Taxonomy`、`PubMed`、`CDD`、`Structure`
5. 标准化后写入本地酶数据库
6. 生成导入报告供用户查看和重试

### 16.2 Match 按钮迭代路线
- 第一版：基于 accession、蛋白名、物种、GeneID、EC 编号做规则召回
- 第二版：增加 PMID/DOI 回查、强弱关联分层和关联依据展示
- 第三版：增加结构域、功能位点、摘要语义信息的综合打分

### 16.3 文献展示边界
- 首期仅展示公开元数据和链接，不分发受版权限制的全文 PDF。
- 对 `references` 自由文本标题不直接信任，必须回查 `PMID/DOI` 标准元数据后再展示。
